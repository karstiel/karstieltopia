      \n
    • Depurar lambdas.<\/strong> Considero que esta opci\u00f3n es muy importante porque aumenta el soporte a Java 8.     • Forzar un return en el modo debug.<\/strong> Si estamos depurando y no queremos que se siga ejecutando el m\u00e9todo en el que estamos, ahora podemos forzar el volver al m\u00e9todo padre sin tener que ejecutar el resto del m\u00e9todo.     • El interfaz gr\u00e1fico de los tests ha sido modificado para mostrar m\u00e1s informaci\u00f3n durante la ejecuci\u00f3n de los tests.<\/strong><\/li>\n<\/ul>\n

      Pero IntelliJ IDEA 15 viene con m\u00e1s novedades que las que he mencionado, por ejemplo viene con soporte para Kotlin <\/a>(soporte beta del nuevo lenguaje para la\u00a0JVM de JetBrains), cambios menores del interfaz gr\u00e1fico de IntelliJ IDEA, novedades en el soporte de los lenguajes de programaci\u00f3n Groovy y Scala; soporte para Spring 4.2 y Spring Boot, novedades de\u00a0Android Studio 1.3<\/a>, un mont\u00f3n de novedades en el soporte de lenguajes y frameworks de frontend, novedades en el soporte de git (operaci\u00f3n rebase, cambiar c\u00f3digo directamente en la ventana de commit, soporte para operaciones de ramas), m\u00e1s soporte para Perforce, mejoras en las herramientas de conexi\u00f3n a base de datos SQL, soporte para docker, etc.<\/p>\n
