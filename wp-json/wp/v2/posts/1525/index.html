Si est\u00e1is intentando ejecutar una m\u00e1quina virtual dentro de VirtualBox <\/a>o VMware <\/a>en Windows y os encontr\u00e1is con un error relacionado con Hyper V es que ten\u00e9is instalado otro motor de m\u00e1quina virtuales, Hyper-V<\/a>, desarrollado por Microsoft, que se instala, por ejemplo cuando se instala el emulador de Windows Phone.<\/p>\n

<\/a><\/p>\n

Si quer\u00e9is arreglar esto, hay que desactivar el Hyper-V, para lo cual ten\u00e9is que abrir una consola con privilegios de administrador y:<\/p>\n

bcdedit \/set \/hypervisorlaunchtype off<\/pre>\n
y para reactivarlo:<\/p>\n
bcdedit \/set \/hypervisorlaunchtype off<\/pre>\n
Sin embargo, si est\u00e1is con Windows 8.1, puede ser que VMware vea que Hyper-V sigue activo, con lo que hay que ser m\u00e1s brusco y hacer lo siguiente:<\/p>\n
dism.exe \/Online \/Disable-Feature:Microsoft-Hyper-V<\/pre>\n
Y para reactivarlo:<\/p>\n
dism.exe \/Online \/Enable-Feature:Microsoft-Hyper-V<\/pre>\n
