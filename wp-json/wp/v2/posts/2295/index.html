Nota 1:<\/strong> Es importante conocer una cosa, todas las versiones de Java son compatibles hacia atr\u00e1s, es decir, si desarrollas algo en Java 5, esto funcionar\u00e1 en Java 6, pero no al rev\u00e9s. Como dije anteriormente deber\u00edas bajar la \u00faltima versi\u00f3n disponible porque aqu\u00ed queremos aprender y normalmente queremos aprender las t\u00e9cnicas m\u00e1s modernas disponibles. En entornos reales, cambiar de versi\u00f3n de Java no es una decisi\u00f3n que se pueda tomar a la ligera y hay que tener cuidado al respecto.<\/p>\n

Nota 2:<\/strong> Si has echado un vistazo a la p\u00e1gina de descargas de Oracle habr\u00e1s visto que hay dos versiones de Java: JDK y JRE. JDK es un acr\u00f3nimo que significa “Java Development Kit” y JRE significa “Jara Runtime Environment”. B\u00e1sicamente, si solo quieres ejecutar aplicaciones Java, solo necesitas instalar el JRE y con ello ya podr\u00e1s ejecutar la m\u00e1quina virtual de Java pero, si quieres desarrollar, necesitas el compilador y otras herramientas que solamente est\u00e1n en el paquete JDK (que tambi\u00e9n incluye todo lo que tiene el JRE).<\/p>\n

Note 3:<\/strong> Puedes usar cualquier versi\u00f3n de Java pero recuerda que Oracle solo mantiene las \u00faltimas versiones.<\/p>\n
